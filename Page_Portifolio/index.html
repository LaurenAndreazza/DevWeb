Perguntas Teóricas

Conceitos fundamentais sobre linguagens de marcação

HTML

O que é Hypertext Markup Language (HTML)?

O HTML é, basicamente, o "esqueleto" de qualquer página na internet. Ele não é uma linguagem de programação (não tem lógica, variáveis ou loops), mas sim uma linguagem de marcação. Nós usamos "tags" como <h1> ou <p> para dizer ao navegador o que é cada parte do conteúdo: título, parágrafo, imagem, link, etc.

História

Como e por quê surgiu o HTML?

O HTML foi criado no início dos anos 90 por Tim Berners-Lee, enquanto trabalhava no CERN (Organização Europeia para a Pesquisa Nuclear).

❓ Por quê? Cientistas precisavam compartilhar documentos de pesquisa entre computadores e sistemas diferentes.
💡 Como? A junção do texto com o "Hipertexto" criou uma teia de informações interligadas — a World Wide Web.
XHTML

O que é eXtensible Hypertext Markup Language (XHTML)?

O XHTML é o HTML reescrito seguindo as regras rígidas do XML. Foi uma iniciativa da W3C para organizar o código da web no início dos anos 2000, exigindo que os desenvolvedores escrevessem código muito mais limpo e rigoroso.

Diferença

Qual a diferença de XHTML para HTML?

A palavra-chave é rigor. O HTML tradicional era permissivo — esqueceu de fechar uma tag? O navegador resolvia. O XHTML não perdoa:

  • Todas as tags devem obrigatoriamente ser fechadas (ex: <br />).
  • Tudo deve ser escrito em letras minúsculas.
  • Atributos devem estar sempre entre aspas.
  • A estrutura deve ser perfeitamente aninhada.
XML

O que é eXtensible Markup Language (XML)?

O XML é uma linguagem de marcação com objetivo totalmente diferente do HTML. Em vez de exibir informações, ele foi criado para armazenar e transportar dados. Você mesmo inventa as tags: <aluno>, <nota>, <curso> — sem regras predefinidas.

Comparativo

Qual a diferença de HTML para XML?

A diferença principal está no foco de cada linguagem:

  • HTML — feito para exibir dados. Tags fixas como <div>, <ul>, <img>.
  • XML — feito para descrever e transportar dados. Você tem liberdade total para criar as próprias tags.
